1) Чтобы скачать файл по данной ссылке, я прописал в терминал команду
$ wget https://sourceforge.net/projects/boost/files/boost/1.69.0/boost_1_69_0.tar.gz
На экране появляется множество различной информации, в том числе процент выполнения операции.

   2) Когда процесс скачивания завершен, я прописал следующую команду для распаковки архива:
$ tar -xf boost_1_69_0.tar.gz
На консоль не выводится ничего.

   3) Затем  я прописал команду 
$ cd boost_1_69_0/
чтобы перейти в только что созданную директорию с распакованным архивом.
Чтобы подсчитать количество файлов, я использовал команду find, которая 
выводит на экран файлы с заданными характеристиками. В конце строки я дописываю команду | wc , которая выводит на экран количество строк, слов и символов. Мне нужно только первое число, которое как раз покажет мне количество файлов в директории (из результата необходимо будет вычесть 1, так как первая строка при выполнении команды find пустая)
Таким образом, команда
$ find -maxdepth 1 | wc
выдает результат 19 19 208, из чего я делаю вывод, что в папке 18 (19-1) файлов.

   4) Чтобы посчитать все файлы в папке, включая вложенные директории, я убираю команду -maxdepth 1 и ввожу просто
$ find | wc
Получаю результат:   66829   66832 3286647, из чего делаю вывод, что всего файлов 66828 (66829-1).

   5) Далее я считал файлы с различными расширениями:
Чтобы найти все заголовочные файлы использую команду:
$ find -name "*.h" |wc
на консоль выводится следующее: 296     296   1173. Так как здесь мы использовали маску - “*.h”, первая строка не является пустой и 1 вычитать из результата мы не будем. Заголовочных файлов в директории ровно 296.

$ find -name "*.cpp" |wc
Данная команда выдает следующий результат:  13774   13774  647953. Значит в директории 13774 файла с расширением .срр.

$ find -not -name "*.h" -not -name "*.cpp" |wc
Эта команда ищет файлы, не являющиеся ни заголовочными, ни файлами с раширением .срр. Результат:  52759   52762 2626956. Но здесь мы должны отнять 1, так как мы искали файлы, не соответстующие маскам, а значит была учтена 1 пустая строка. Итак, искомых файлов в директории 52758.

   6) Дальше я искал полный путь дь файла  any.hpp, снова используя команды find, wc и маску:
$ find -name any.hpp
На экран выводятся расположения нескольких файлов с таким же названием, но нам нужен именно тот, который лежит в директории boost:   ./boost/any.hpp
То есть полный путь искомой папки: ~/boost_1_69_0/boost/any.hpp

   7) Чтобы найти все файлы, в которых упоминается последовательность boost::asio, будем использовать команду grep, которая осуществляет поиск внутри файлов. Чтобы искать данное упоминание также и во всех вложенных директориях, дописываем параметр -r.
$ grep -r boost::asio
На экран выводится большое количество файлов, соответствующих нашим требованиям.

8) Далее я осуществляю компиляцию, как сказано в инструкции:
Перехожу в директорию tools/build/, используя команду $ cd tools/build/, после чего ввожу команду ./bootstrap.sh, чтобы запустить нужный файл. Далее прописываю команду
$ ./b2 install –prefix=rezult, чтобы скомпилировать нужные нам файлы.
На консоль выводится информация о компилирующихся файлах.

9) Затем мы переносим скомпилированные файлы в нужную директорию.
$ mkdir ~/boost-libs (создаем нужную директорию)
$ mv rezult/ ~/boost-libs/ && cd && cd boost-libs (переносим файлы в директорию,
и переходим в нее)

10) Чтобы подсчитать вес каждого файла этой директории ввожу команду
$ find . -type f -exec ls -s {} \;
Команда find находит все файлы в данной директории.
С помощью опции -exec мы каждый найденный файл перенаправляем в команду ls, которая с помощью опции -s выводит нам подробную информацию о данном файле, нас интересует размер файла.
На экран выводятся названия файлов и их вес, например: 
16 ./rezult/share/boost-build/src/tools/stage.py
28 ./rezult/share/boost-build/src/tools/doxygen.jam
8 ./rezult/share/boost-build/src/tools/rc.py
8 ./rezult/share/boost-build/src/tools/symlink.jam
4 ./rezult/share/boost-build/src/tools/builtin.jam
4 ./rezult/share/boost-build/src/tools/cast.py
0 ./rezult/share/boost-build/src/tools/__init__.py

11) Чтобы найти 10 самых тяжелый, прописываю команду
$ find . -type f -exec ls -s {} \; | sort -n | head -10


Команда sort -n сортирует переданный ей список файлов по первому числу, что является размеров файла, и передает результат дальше.
Команда head -10 выводит нам только 10 первых пунктов переданных ей на вход данных.

* ./rezult/include/boost/accumulators/framework/accumulator_base.hpp
